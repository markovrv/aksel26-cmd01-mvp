## Доработанное техническое задание (ТЗ) на MVP платформы «Индустриальный гид»

### 1. Цель MVP (уточнение)

Создать работающий прототип B2B2C-платформы, который позволяет промышленным предприятиям Кировской области через **интерактивную карту** привлекать студентов для решения реальных производственных задач (кейсов), а студентам — находить эти задачи, отправлять решения и получать приглашения на стажировки и экскурсии.

**Ключевое отличие от исходного ТЗ:** Карта — это не просто справочник, а инструмент навигации по **возможностям трудоустройства и развития**.

---

### 2. Пользовательские роли и их задачи (расширено)

| Роль | Главная задача в MVP |
|------|----------------------|
| **Студент** | Найти интересный кейс/стажировку на карте или в каталоге, отправить решение, получить обратную связь от предприятия. |
| **Представитель предприятия (HR/инженер)** | Разместить кейс/экскурсию/стажировку, посмотреть отклики студентов, скачать решения, пригласить лучших. |
| **Администратор платформы** | Добавлять/одобрять предприятия, модерировать кейсы, смотреть общую статистику (студенты, решения). |

---

### 3. Детальное описание сущностей (ядро MVP)

Без чётких сущностей разработчик начнёт додумывать сам — это риск. Ниже — то, что нужно хранить в базе данных.

#### 3.1. Сущность «Пользователь»
- ID
- Роль (student / company / admin)
- Email (уникальный, используется для входа)
- Пароль (хеш)
- Статус (active / blocked)
- Дата регистрации

#### 3.2. Сущность «Профиль студента» (расширение пользователя)
- Имя, фамилия
- Вуз / колледж
- Курс / год обучения
- Направление / специальность
- Город
- Телефон (опционально)
- Ссылка на резюме (PDF) — *можно загрузить файл*
- Список отправленных решений (связь с сущностью «Решение»)

#### 3.3. Сущность «Профиль предприятия»
- Название (полное)
- Краткое описание (до 500 символов)
- Город, адрес (для отображения на карте)
- Координаты (широта, долгота)
- Логотип (изображение)
- Контактное лицо (ФИО)
- Телефон, email (для связи)
- Статус модерации (moderation / active / rejected)

#### 3.4. Сущность «Кейс»
Используется для кейсов.

- ID
- Предприятие (связь)
- Название
- Описание (полное)
- Требования к студенту
- Дедлайн приёма решений (дата)
- Что прикреплено:
  файл с заданием (PDF/Word)
- Статус (draft / active / closed / archived)

#### 3.5. Сущность «Решение»
- ID
- Студент (связь)
- Кейс (связь)
- Текст решения (до 2000 символов)
- Файл решения (PDF, DOCX, ZIP, не более 10 МБ)
- Дата отправки
- Статус рассмотрения:
  - `new` (отправлено)
  - `viewed` (просмотрено)
  - `invited` (принято)
  - `rejected` (отклонено)

#### 3.6. Сущность «Уведомление» (упрощённо, без очередей)
- ID
- Пользователь (кому)
- Текст
- Прочитано (да/нет)
- Ссылка на связанный объект (кейс, решение)

### 4. Детальный пользовательский опыт (User Story)

#### 4.1. Сценарий: Студент

**Шаг 1. Главная страница → карта**
- Студент видит карту с маркерами предприятий Кировской области.
- Маркеры цветные: синий (есть активные кейсы), серый (кейсы не загружены), зеленый (студент уже отправил решение кейса предприятия ).
- Слева — панель фильтров:
  - Поиск по названию
  - Тип: Кейсы / Стажировки / Экскурсии
  - Отрасль (список из админки)
- При клике на маркер — всплывает карточка: название, логотип, количество активных кейсов, кнопка «Подробнее».

**Шаг 2. Страница предприятия (публичная)**
- Описание, контакты, фото/видео.
- Вкладки: «Кейсы», «Стажировки», «Экскурсии».
- Студент видит только **активные** мероприятия с ненаступившим дедлайном.
- Кнопка «Отправить решение» (для кейса) или «Подать заявку» (для экскурсии/стажировки).

**Шаг 3. Отправка решения по кейсу**
- Форма:
  - Текстовое поле «Ваше решение» (обязательно)
  - Загрузка файла (PDF, DOCX, до 10 МБ)
  - Кнопка «Отправить»
- После отправки:
  - Показываем подтверждение.
  - Решение появляется в личном кабинете студента (раздел «Мои отклики»).

**Шаг 4. Личный кабинет студента**
- Мои отклики (список кейсов/заявок) со статусами:
  - Новая
  - Просмотрена
  - Приглашён (показываем контакт предприятия)
  - Отклонён
- Возможность скачать свой файл решения.
- Редактирование профиля (вуз, курс, резюме).

---

#### 4.2. Сценарий: Представитель предприятия

**Шаг 1. Регистрация и профиль**
- Регистрируется, ждёт подтверждения админа (в MVP — вручную).
- После активации заполняет профиль: название, логотип, описание, адрес, координаты на карте.

**Шаг 2. Создание кейса (самое важное!)**
- Форма:
  - Тип: Кейс
  - Название задачи
  - Описание (проблема, контекст)
  - Требования к студенту (навыки, курс)
  - Загрузить файл с заданием
  - Дедлайн приёма решений
- После создания кейс попадает на модерацию (админ включает публикацию).

**Шаг 3. Просмотр решений**
- В ЛК предприятия раздел «Мои кейсы → Решения».
- Таблица:
  - Студент, вуз, дата отправки
  - Ссылка скачать файл
  - Текстовое решение (показать на модалке)
  - Действия: «Пригласить», «Отклонить», «Просмотрено»
- При изменении статуса студенту приходит уведомление (в интерфейсе).

---

#### 4.3. Сценарий: Администратор

**Шаг 1. Вход в админ-панель**
- Доступен только для роли admin.

**Шаг 2. Модерация предприятий**
- Список новых регистраций.
- Возможность: одобрить, отклонить, отредактировать данные.

**Шаг 3. Модерация кейсов**
- Список кейсов от предприятий.
- Возможность: опубликовать, отклонить с комментарием, отредактировать текст.

**Шаг 4. Управление пользователями**
- Блокировка / разблокировка, редактирование студентов, решений и компаний, кейсов.
- Просмотр количества зарегистрированных студентов, кейсов, решений.

### 5. Что должно работать в MVP (чёткий список)

| № | Функция | Кто использует |
|---|---------|----------------|
| 1 | Регистрация и выбор роли (студент / предприятие) | Все |
| 2 | Вход по email + пароль | Все |
| 3 | Просмотр карты с маркерами предприятий | Гость / студент |
| 4 | Фильтрация по типу (кейсы / стажировки / экскурсии) | Студент |
| 5 | Просмотр страницы предприятия и списка его кейсов | Студент |
| 6 | Отправка решения (текст + файл) по кейсу | Студент |
| 7 | Просмотр своих решений и их статусов | Студент |
| 8 | Создание кейса (описание, требования, файл, дедлайн) | Предприятие |
| 9 | Просмотр решений по своему кейсу (список, скачать файл) | Предприятие |
| 10 | Смена статуса решения (новая → приглашён / отклонён) | Предприятие |
| 11 | Админ-панель: одобрение предприятий и кейсов | Админ |
| 12 | Админ-панель: блокировка пользователей | Админ |
