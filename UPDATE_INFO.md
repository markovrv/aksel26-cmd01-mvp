# 📋 Обновление проекта - Полная версия MVP

## ✅ Что было добавлено

### 1. **Русский язык интерфейса**
- Весь интерфейс переведен на русский язык
- Все тексты, кнопки, метки и сообщения на русском

### 2. **Интерактивная карта**
- Интеграция Leaflet с OpenStreetMap
- Маркеры предприятий с информацией о мероприятиях
- Боковая панель с фильтрами и поиском
- Клик на предприятие центрирует карту

### 3. **Расширенная модель данных**
- **Студенты** - отдельная таблица с профилем
- **Компании** - отдельная таблица со статусом модерации
- **Мероприятия** - кейсы, стажировки, экскурсии (единая сущность)
- **Заявки** - система откликов с 4 статусами (новая, просмотрена, приглашена, отклонена)
- **Логи** - система логирования для админ-панели

### 4. **Типы мероприятий**
- 📋 **Кейсы** - задачи от предприятий с решением
- 🎓 **Стажировки** - программы обучения
- 👁️ **Экскурсии** - посещение производства

### 5. **Фильтры и поиск**
- Поиск по названию мероприятия
- Фильтр по типу (кейс/стажировка/экскурсия)
- Фильтр по городу
- Список предприятий с синхронизацией на карте

### 6. **Профили пользователей**
- **Студент**: ФИО, ВУЗ, курс, специальность, город, телефон
- **Компания**: название, ИНН, город, сайт, контактное лицо, описание, логотип
- **Админист**: управление компаниями и модерация

### 7. **Статусы модерации**
- Компании: на модерации → активна → отклонена/заблокирована
- Мероприятия: активное, закрытое, скрытое
- Заявки: новая → просмотрена → приглашена/отклонена

### 8. **Админ-панель**
- 📊 Статистика: студенты, компании, мероприятия, заявки
- 🏢 Управление компаниями: одобрение/блокировка
- 📜 Логи системы: отслеживание действий пользователей

## 📁 Структура базы данных

### Таблицы
- `students` - профили студентов
- `companies` - профили компаний (со статусом модерации)
- `events` - мероприятия (кейсы, стажировки, экскурсии)
- `applications` - заявки на мероприятия
- `admins` - администраторы
- `logs` - логи действий
- `sessions` - активные сессии пользователей

## 🎯 Главные сценарии использования

### Для студента
1. Регистрация → выбор роли "Студент"
2. Просмотр карты с предприятиями
3. Просмотр мероприятий (кейсы, стажировки, экскурсии)
4. Отправка заявки на мероприятие
5. Отслеживание статуса заявки в личном кабинете

### Для компании
1. Регистрация → выбор роли "Предприятие"
2. Ожидание одобрения админист­ратора
3. После одобрения: создание мероприятий
4. Просмотр заявок от студентов
5. Изменение статуса заявок (просмотрена, приглашена, отклонена)

### Для администратора
1. Вход в админ-панель
2. Просмотр статистики
3. Управление компаниями (одобрение/блокировка)
4. Просмотр логов системы

## 📊 API эндпоинты

### Аутентификация
```
POST   /api/auth/student/register
POST   /api/auth/student/login
POST   /api/auth/company/register
POST   /api/auth/company/login
POST   /api/auth/admin/login
POST   /api/auth/logout
GET    /api/auth/me
```

### Мероприятия
```
GET    /api/events              (с фильтрами)
GET    /api/events/:id          (подробнее)
POST   /api/events              (создание, компания)
```

### Заявки
```
POST   /api/applications        (отправка)
GET    /api/student/applications
GET    /api/company/applications
PUT    /api/applications/:id    (изменение статуса)
```

### Профили
```
GET    /api/company/:id
GET    /api/student/dashboard
GET    /api/company/dashboard
```

### Админ
```
POST   /api/admin/login
GET    /api/admin/companies
PUT    /api/admin/company/:id/status
GET    /api/admin/stats
GET    /api/admin/logs
```

## 🌟 Особенности реализации

1. **Карта на Leaflet** - легко расширяется, не требует API ключей
2. **Модульная архитектура** - легко добавить новые типы мероприятий
3. **Полная система логирования** - отслеживание всех действий
4. **Система статусов** - гибкое управление жизненным циклом мероприятий
5. **Фильтрация и поиск** - быстрый поиск нужных мероприятий
6. **JWT аутентификация** - безопасная работа с сессиями

## 🔄 Статусы мероприятий

- **active** - активное, принимает заявки
- **closed** - закрыто, уже не принимает новых заявок
- **hidden** - скрыто от студентов
- **pending** - на модерации (только для компаний)

- Аутентификация: JWT + Sessions (безопасно и масштабируемо)
