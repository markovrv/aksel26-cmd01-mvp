| Сервис       | Контейнер                  | Порт | Доступность |

## Как работает API проксирование

1. **Пользователь** → `http://localhost:21426/api/events`
2. **Nginx** → перенаправляет запрос на `http://server:5000/api/events`
3. **Backend** → сериюает JSON → возвращает клиенту

Это позволяет:
- ✅ Избегать проблем CORS
- ✅ Скрыть бэкенд от прямого доступа
- ✅ Объединить фронтенд и API на одном порту

## Команды

```powershell

docker-compose up -d

# Остановить
docker-compose down

# Перезапустить
docker-compose restart

# Логи
docker-compose logs -f

# Изменить/обновить конфигурацию
docker-compose build --no-cache
docker-compose up -d

# Полный сброс (удалить все данные)
docker-compose down -v
```
